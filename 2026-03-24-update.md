# wtfoc — Day 2 Status Update

**Today we turned wtfoc into a multi-source knowledge graph with an MCP server, Claude Code skills, and a validated cross-stack demo.**

You can now:

- **Ingest documentation sites** (crawlee + turndown) — crawled docs.filecoin.cloud (94 pages, 2,848 chunks) in 7 seconds
- **Trace a feature across 5 source types** — SDK code, GitHub issues, PRs, docs, and markdown from multiple repos in one query
- **Use wtfoc from Claude Code natively** — MCP server exposes trace, query, ingest, status as tools
- **Run `/trace-analyze`** — Claude Code skill that traces a question and synthesizes a structured analysis with evidence chains and gap detection
- **Run `/drift-check`** — finds stale docs by comparing GitHub activity against documentation content
- **Resume interrupted ingests** — chunk dedup skips already-stored content automatically
- **Ingest large repos without OOM** — batched pipeline processes 500 chunks at a time

**Demo highlight**: Traced the full file upload flow across synapse-sdk, filecoin-pin, curio, filecoin-services, and docs.filecoin.cloud — one query mapped the path from `StorageManager.upload()` through the 3-phase pipeline to PDP verification on the SP side. Found 3 open issues along the way. See `docs/demos/upload-flow-trace/`.

# Thread: what shipped today

## Features

**Adapter registry (#53)**
- Replaced 210-line if/else chain with pluggable Map-based registry
- New adapters self-register at module load, CLI resolves by name
- Net -68 lines

**Website adapter (#32)**

- 18 new tests

**Valibot schema validation (#38)**

- All 25 existing schema tests pass without modification

**MCP server (#66)**
- `@wtfoc/mcp-server` with 5 tools: `wtfoc_trace`, `wtfoc_query`, `wtfoc_ingest`, `wtfoc_status`, `wtfoc_list_sources`
- Stdio transport, long-lived process, embedder initialized once

**Claude Code skills (#63)**
- `/trace-analyze` — trace + auto-expand gaps + synthesize structured report
- `/collection-setup` — interactive guide to set up a collection from a project description
- `/drift-check` — compare GitHub activity against docs to find stale pages

## Infrastructure

**Batched ingest pipeline**
- Process chunks in configurable batches (default 500) instead of collecting all in memory
- Fixed OOM on large repos (synapse-sdk: 5,179 chunks in 26 batches)
- `--batch-size` flag for tuning

**Resumable ingest with dedup**
- Builds set of known chunk IDs from existing segments
- Skips duplicates on re-run — makes ingest idempotent and crash-recoverable
